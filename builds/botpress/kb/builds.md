## Categories

Builds are organised by functional category, shown as filter chips on this page rather
than as items in the top navigation. Categories currently in use: **Websites**. The full
category set includes chatbots, video, websites, automation, writing, research, design
and agents — a category only appears once it has a real build in it.

## Freshness states

Each build shows a coloured dot and a verified date so you can tell how current it is:

- **Verified recently** (green) — checked and current.
- **Getting old — recheck due** (amber) — time to re-verify; treat the details as
  possibly stale.
- **Archived** (grey) — kept as history; no longer current, and not to be treated as
  working.

Perishable facts (tool versions, pricing, the last-verified date) are shown in a dated
stamp on each build so you can see how fresh a claim is. A build log never implies an old
build still works just because the page still exists.
